EAS Tools - TTS Documentation

Text-to-Speech (TTS) Documentation

The Text-to-Speech (TTS) feature in EAS Tools allows users to convert text messages into spoken audio. This can be particularly useful for generating alerts with announcements based on the text input.

How to Use the TTS Feature

  1. Navigate to the encoder section of the EAS Tools application.
  2. Enter the text you wish to convert into speech in the provided text area after checking "Spoken message".
  3. Select the desired voice.
  4. Click the "Generate" button to create the speech audio alongside the tones.
  5. Listen to the generated audio or download it for later use.

Supported Voices

EAS Tools supports a variety of voices for text-to-speech conversion. Users can choose from a list of different voices to use, each with their own unique characteristics. The following is a list of voices currently available to users, and what backends they use in square brackets:

  • [WASM] WebAssembly (local) Voice

NOTE: Except for the WebAssembly voice, all voices listed here are processed by an outside server located at Wags' Puzzle Space. This is because they require server-side processing to generate the speech audio, which is beyond the capabilities of a simple GitHub Pages site.

Voice Descriptions

Each voice has its own unique sound and characteristics. For example, the DECTalk voices are known for their distinctive robotic sound, while the other voices offer a more natural-sounding speech synthesis. Users can experiment with different voices to find the one that best suits their needs. If you want a small taste of each voice, you can listen to the demos provided on the demos page.

Additional Notes

When using the TTS feature, users should be aware of the following:

  • The quality of the generated speech may vary depending on the selected voice. See below for pronunciation notes and examples.
  • There is a limit of 20 successful requests per day for the server-side voices to prevent abuse. This limit resets daily at midnight Central Time. All processed audio is also cleared at this time, so make sure you download your audio before then.
  • Each TTS request is allowed up to 90 seconds to complete itself. If a request takes longer than this, it will be terminated and abandoned by the server. Most requests should complete LONG before this timer is up, but know that you may need to adjust your text input or try a different voice if you encounter timeouts. Timeouts usually result in an "x-empty" response by the TTS server.

Pronunciation Notes and Examples

Here are some pronunciation notes and examples for the different voices available in EAS Tools:

  • Each voice will pronounce each word in the text input according to its own unique characteristics. Users are encouraged to experiment with different voices to find the one that best suits their needs. YouTube is a great example of where to find pronunciation examples for various text-to-speech voices.
  • Some voices may have difficulty pronouncing certain words or phrases, especially if they are uncommon or technical in nature. Users may need to adjust their text input to ensure that the generated speech is clear and understandable. See below for some codes you can use to adjust pronunciation depending on the backend used for the given voice.
  • If you find yourself running into the daily request limit frequently, try installing some of the voices locally so you know what to expect when using them. This also helps reduce server load since you're using the voices locally instead of relying on an outside service. Installers for most of the voices can be found on websites like the Internet Archive or the "EAS Mock Resources" Carrd site.

Here are some specific pronunciation codes for different backends:

  • [WASM] WebAssembly: Does not apply, since the voice would need further training for specific phrases/words. There is no built-in way to modify pronunciation or speed with WebAssembly voices.
  • [VT] VoiceText/NeoSpeech: See the detailed documentation PDF for a full list of examples and documentation for the VTML format. A complete example of VoiceText is below for demonstration purposes.
  • [BAL] Balabolka/Balcon: You can use the SAPI/Microsoft Speech Platform codes to adjust things like silence periods, speech rate (how fast the text is spoken), speech pitch, volume, and phonemes (the big one). Phonemes are formatted like so: <pron sym="t eh - 1 k ey - m ax"/>, which would pronounce the word "Tekamah" (city in Nebraska) correctly.   • [DT] DECTalk: DECTalk has its own set of phoneme codes that can be used to adjust pronunciation. An example would be [:phoneme on] [axs'owshiyeyt]. This will pronounce the word "associate" like "uh-soh-shee-ayt" (to associate something with someone or something), not like "uh-soh-shee-at" (an associate/worker at a store). A full list of DECTalk phoneme codes can be found in the DECTalk documentation.
  • [FEST] Festvox: Festvox uses the Festival Speech Synthesis System, which has no way to override the pronunciation of words "on the fly" like other engines. You're limited to the pre-defined pronunciations in the voice model.
  • [EMNet] EMNet: The EMNet voice uses a bunch of pre-recorded audio files and just combines them according to the header and time the alert expires, so there are no options (other than what timezone you wish to use).
  • [BAL/SPFY] Speechify: Speechify uses the same phoneme codes as Balabolka/Balcon, so you can use the same techniques above to adjust pronunciation. Note, however, that the requests to Tom will take longer depending on text length. Avoid sending ultra-long text through Tom, or your request may time out entirely.

If you're having trouble with pronunciation of specific words, try adjusting the phoneme codes or using different text input that sounds more literal to the word you're trying to emulate (e.g., "teh kay mah" vs "Tekamah"). You can also experiment with different voices to see if they handle the text better. If you need an exact phoneme for a specific word and voice, try the TTS Phoneme Helper GPT, which is built and trained on the data in this documentation. (NOTE: Requires a ChatGPT account to use.)
