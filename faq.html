What is the EAS?

The Emergency Alert System (EAS) is a national public warning system used in the United States that allows authorized government officials to send emergency alerts and warnings to the public via radio, television, and other communication channels. Learn more at Wikipedia.

What is EAS-Tools?

EAS-Tools is a web-based application designed to help users decode and encode EAS SAME headers, generate text crawls, and splice audio files for EAS alerts. It provides a user-friendly interface for working with EAS data and is provided for lab, testing, hobbyist, and educational purposes only. The author is NOT responsible for ANY misuse.

What is SAME?

SAME (Specific Area Message Encoding) is a protocol used in the EAS to encode information about emergency alerts, including the type of alert, affected areas, and duration. SAME headers are transmitted as part of the EAS message and are used by receiving devices to determine whether to activate and display the alert. You can most commonly find SAME technology in weather radios and some newer TVs using the ATSC 3 standard. The SAME tones are those 3 repeat distinctive audio signals that accompany the alert messages before and after the alert announcement (if any, usually Required Weekly Tests skip the announcement).

What is AFSK modulation?

AFSK (Audio Frequency-Shift Keying) modulation is a method of encoding digital data using audio tones of different frequencies. In the context of the EAS, AFSK modulation is used to transmit SAME headers as audio signals that can be decoded by receiving devices. The EAS SAME tones use AFSK modulation at 520.83 bits per second, with a mark frequency of 2083.3 Hz and a space frequency of 1562.5 Hz. You only really need to know this information if you're making your own encoder/decoder, but since this repository provides the necessary tools to work with AFSK modulation in the context of the EAS, there's no real need to know this info (unless you want to be a developer).

What is the purpose of the EAS-Tools web application?

The purpose of the EAS-Tools web application is to provide a convenient and accessible way for users to work with EAS SAME headers and related data. It allows users to decode and encode SAME headers, generate text crawls, and splice audio files for EAS alerts, all within a web browser. This makes it easy for hobbyists, educators, and developers to experiment with EAS data without needing complicated, usually hard to use, specialized software or hardware. Anyone can freely use the application without any cost or licensing fees.

How does EAS-Tools work?

EAS-Tools is a web-based application that allows users to decode and encode EAS SAME headers directly in their web browser, among other features. It uses JavaScript to process the data locally, ensuring that no data is sent to external servers (except for web TTS requests, more on that below). No analytics or tracking is performed on user data. Any web TTS requests are only logged until Midnight Central Time, for debugging purposes only, and the logs and generated audio are automatically deleted after that time from the server. Your privacy is important to me, and I am committed to protecting it as best as I know how.

You mentioned a web TTS service? What's that, and how does it work?

The web TTS (Text-to-Speech) service is a server-side component that generates speech audio from text input. When you use the EAS-Tools Encoder or Audio Splicer tools and select the Text-to-Speech announcement option and a voice other than WebAssembly/NanoTTS, your text is sent to the TTS server located at Wags' Puzzle Space. The server processes the text and returns the generated audio back to your browser for inclusion in the EAS alert. This allows for dynamic generation of speech audio without relying on client-side TTS capabilities, which are extremely limited when it comes to being able to synthesize real voices used in actual alerts. Please note that any/all requests made to the TTS server are logged temporarily for debugging purposes only, and all logs and generated audio are automatically deleted at exactly Midnight Central Time. Each individual user is limited to requests per day overall to prevent abuse of the service. If you find yourself running into the request limit often, try installing some of the voices you're looking to synthesize on a regular basis and using them locally! The web TTS service is provided AS-IS, with no guarantees or warranties, or uptime promises. It may go down randomly, it may not. The server tries to check and make sure it is up before it processes audio, but if it happens to suddenly, randomly go down in the middle of this processing, it may not be able to complete your request successfully. Keep this in mind when using the Web TTS service.

Getting Started with EAS-Tools

To get started with EAS-Tools, simply navigate to the main page and select the tool you wish to use from the available options in the top right navigation section. Each tool has its own dedicated section (below) with detailed instructions on how to use it effectively and efficiently.

Understanding SAME Header formatting

SAME headers follow a specific format that includes various fields such as Originator, Event Code, Location Codes, Valid Time Period, and sender ID. Familiarizing yourself with this format is essential for effectively using the Decoder and Encoder tools. Summed up as quickly as possible, a SAME header consists of a series of hyphen-separated values that convey important information about the alert. A sample SAME header looks like the following:

ZCZC-PEP-NPT-000000+0100-0011337-ORIGIN-

Each segment of the header represents specific information about the alert, such as the type of event, the location it pertains to, and the time it was issued. Let's break it down into its parts, one by one, and go over what each one does or means:

  • ZCZC: This is the start of the SAME header, indicating the beginning of the message. This is what encoders and decoders look for when processing the alert.
  • PEP: The Originator code, indicating who issued the alert (e.g., PEP for Primary Entry Point). The only options are PEP, EAS (Emergency Alert System Participant, usually a sole TV/AM/FM station), CIV (Civil Authorities), and WXR (NOAA Weather Radio).
  • NPT: The Event Code, specifying the type of alert (e.g., NPT for National Periodic Test). There's a whole list of event codes on Wikipedia.
  • 000000: The Location Code (or codes, if there are multiple), representing the areas affected by the alert. There can be between 1 and 31 location codes.
  • +0100-0011337: The Valid Time Period, indicating how long the alert is valid for (duration and issuance Julian date/UTC time).
  • ORIGIN: The Originator Identifier, providing additional information about the source of the alert. This is usually where the station's call sign or other identifying information would be included.
  • -: The end of the SAME header, indicating the conclusion of the message.

So, in the example above, we have a SAME header that indicates a National Periodic Test (NPT) issued by a Primary Entry Point (PEP) for the whole of the United States (000000), valid for 1 hour from 13:37 UTC on January 1 of the issuance year. IMPORTANT NOTE: SAME does NOT specify a year in the header! It simply assumes that all alerts are issued for the current year. EAS-Tools follows this assumption and automatically uses the current year when processing SAME headers.

Tool: Decoder

The Decoder tool allows you to decode EAS SAME headers, either from real-time audio via your computer's microphone inputs, or via a pre-recorded audio file.

To use the Decoder tool, follow these steps:

  1. Select the "Start Decoder" button to decode live audio from your microphone. Otherwise, select "Upload Audio File" to upload a pre-recorded audio file.
  2. Wait for an alert to come in on whatever microphone input you chose, or wait for your file to be processed by the JavaScript decoder.
  3. Once a SAME header is detected, it will be displayed in the output section alongside a "View Alert" button.
  4. Click the "View Alert" button to see detailed information about the decoded SAME header.
  5. (optional) Click "Clear Logs" to clear the output section.

The output section will display the decoded SAME header information, including all relevant fields and their values. This can be useful for verifying the accuracy of the decoding process and your own personal knowledge of the SAME protocol.

Tool: Encoder

The Encoder tool allows you to create full EAS alerts by filling out a form with the necessary information. Once the form is completed, you can generate a valid alert for lab, testing, hobbyist, and educational purposes only.

To use the Encoder tool, follow these steps:

  1. Choose your header generation mode. If you choose "Build your own", you can customize every part of the generated alert. If "Custom header" is chosen instead, you can only paste an existing SAME header into the form.
  2. Fill out the form with the required information, including the event type, originator, effective time, start date/time, sender ID, and locations.
  3. Choose some settings related to the generated audio, such as the type of attention tone used, how long the attention tone should last, extra marks (for SAGE-sounding encoding), extra spaces (for NWR encoding), and if you want the audio clipped (for authenticity).
  4. Select an announcement type, if any. The default is "No Announcement". You can choose that, "Text-to-Speech Announcement", or "Custom Audio Announcement".
  5. If you chose "Text-to-Speech Announcement", enter the announcement text in the provided text area. If you chose "Custom Audio Announcement", upload your own audio file.
  6. Click the "Generate" button to create the EAS alert audio. If you chose to use the TTS web service, the audio will be generated for you by the server located at Wags' Puzzle Space. This process can either be extremely fast, or take up to 90 seconds (maximum) depending on a number of factors, like the voice used, how long your text is, things like that.
  7. Wait for the audio to be generated. Once complete, a "Save as wav file" button will appear, and you can play/preview the audio in-line with the audio player below the generate/save buttons.
  8. (optional) Click "Clear Logs" to clear the log section.

The generated alert can then be downloaded as a simple WAV file to your local computer for later use. It is important to reiterate that this tool is intended for lab, testing, hobbyist, and educational purposes only. The author is NOT responsible for ANY misuse.

Tool: Text Crawl Generator

The Text Crawl Generator tool allows you to create scrolling text crawls that can be used in conjunction with EAS alert audio. Note, however, you must generate the audio and crawl separately and combine them using an outside tool like ffmpeg or a standard video editor, you can NOT combine them with this tool alone. You can customize pretty much any aspect of the crawl that is generated to fit your needs or scenario.

To use the Text Crawl Generator tool, follow these steps:

  1. Select a crawl text source. You can choose to enter your own text, or use a SAME header. SAME headers pull the alert text using a JavaScript port of the Python library EAS2Text.
  2. If you chose "Custom Text", enter your desired crawl text in the provided text area. If you chose "Generate from EAS Header", enter a valid SAME header in the input field.
  3. The following aspects of the crawl can be customized for maximum effect: Speed (how fast the crawl scrolls), "VDS Mode" (see here for an example of VDS Mode in action in a real alert, specifically noting how the letters "pop up" like a typewriter), font size/family/style (bold/italic/bold italic, or normal), crawl width/height/inset (how far the text is inset from the edges), top left location (if you need the crawl to be dynamically located on a custom screen, for example), restart delay (how long to wait before restarting the crawl), and export repetitions (how many times to repeat the crawl in exports, the live display loops infinitely), text color, background mode (select from "Solid, Transparent, Custom Image, or from a pre-defined image library included with EAS-Tools), and finally, text outline color/width/join style. Whew!
  4. Click the "Start Crawl" button to create the text crawl.
  5. Wait for the crawl to be generated. Once complete, a preview of the text crawl will be displayed below the button controls.
  6. Click "Pause Crawl" to pause the text crawl in place. Use the "Previous Frame" or "Next Frame" controls to go frame-by-frame. Click "Stop/Reset" to reset the crawl position to the start. Click "Export as GIF" or "Export as video (.webm)" to save your crawl to a file, GIF and WebM are supported. You can also click "Copy to Clipboard" to copy the crawl text to your clipboard (especially helpful for E2T crawl text).
  7. (optional) Click "Clear Logs" to clear the log section.

The generated text crawl can then be used in conjunction with EAS alerts or for other purposes as needed.

Tool: Audio Splicer

The Audio Splicer tool allows you to splice together multiple audio files into a single audio file. This can be useful for creating custom EAS alerts that include multiple segments, or customizing an alert to use different voices within the alert.

To use the Audio Splicer tool, follow these steps:

  1. Choose whether to load an audio file, add TTS, or use both interchangeably at will.
  2. If you chose to load an audio file, click the "Load File" button to select and upload your desired audio file(s). You can upload multiple files if needed.
  3. If you chose to add TTS, enter your desired text in the provided text area, select a voice from the dropdown menu, and click the "Generate TTS" button to create the TTS audio segment.
  4. Repeat steps 2 and 3 as needed to add as many audio segments to the timeline as needed.
  5. Once all desired audio segments have been added, arrange them however you see fit using the timeline controls below the waveform preview and other button controls. You can split the selected audio into left/right (selected/unselected), join all segments into one singular segment, trim to selection, delete selection, and save selection as a independent segment.
  6. You can then preview individual segments, the currently selected segment in the waveform editor, and the whole audio file within the interface. You are also able to rename sections and move them up and down in the timeline, or remove them entirely.
  7. You can also insert silence as needed, adjust the loudness of the entire audio, and use macros! Macros determine how the audio will sound after some choice post-processing. By default, no macro is selected, but you can only add one at a time. To chain macros, export and re-import your audio. You can also add static noise as needed to your macro for maximum effect.
  8. Click the "Export WAV" button to create the final spliced audio file with NO MACROS. If you wish to have macros, instead click "Export WAV with Macro".
  9. Once you have exported your audio, it will automatically download it to your local computer for later use.
  10. (optional) Click "Clear Project" to clear the ENTIRE project and start over. YOU WILL LOSE ALL YOUR PREVIOUS PROGRESS!

The spliced audio can then be downloaded as a single WAV file to your local computer for later use. Projects are saved on any edit action, and are stored locally, on your device, so you can resume your previous session with ease.

Tool: Phoneme Converter/Generator

The Phoneme Converter/Generator tool allows you to convert text into phonemes, and phonemes back into their respective word(s). This can be useful for generating custom pronunciations for the web TTS voices that support phoneme inputs. If you have a phoneme that is not generated correctly, or would like to see changed to something more "official", please submit an issue or pull request on the GitHub repository. There is a list of overrides that are populated with some, but not all, phoneme mappings used by organizations like the National Weather Service. You can view the current list of overrides at this link. The "national dictionary" overrides come from the NWR Reference Discord server.

To use the Phoneme Converter/Generator tool, follow these steps:

  1. Enter the text you wish to convert into phonemes in the provided text area.
  2. Click the "Convert to Phonemes" button to generate the phoneme representation of the text.
  3. The generated phonemes will be displayed in the output section below the button.
  4. You can then copy the generated phonemes to your clipboard for use with the Web TTS service or your own local copy of whatever TTS voice(s) you have available.

The generated phonemes can then be used for the express purpose of customizing TTS word pronunciations to your liking.

Tool: Video/Audio Muxer

The Video/Audio Muxer tool allows you to combine video and audio files into a single multimedia file. This can be useful for creating custom alerts that include both video and audio components.

To use the Video/Audio Muxer tool, follow these steps:

  1. Click "Load FFmpeg" to load in either multi-core or single-core FFmpeg. By default, multi-core is preferred for better performance. The loader will only fall back to single-core if the multi-core environment is not supported (which is unlikely).
  2. Select the video and audio files you want to combine using the file input controls.
  3. Optionally, check "Transcode output to MP4" if you want an .mp4 file, otherwise, it will use the extension/container your video file already has (e.g., .mkv, .avi, .webm).
  4. Click the "Append audio to video" button to start the muxing process. The process may take a long time, depending on a number of factors, such as your machine's processing power, if FFmpeg is in multi-core mode, etc. Progress is displayed and shows where in the process the mux is at. As well, you can cancel an in-progress mux by clicking "Cancel" at any time.
  5. Once the muxing process is complete, a "Download result" button will appear. Click this button to download the combined video/audio file. You may also choose to preview the muxed file by using the video controls on the page below the "Append audio to video" button.

The muxed video/audio file can then be used for various purposes, such as sharing your custom alerts with both audio and video combined, without the use of an external tool or full video editor to make it happen.

Troubleshooting

If you encounter any issues while using EAS-Tools, here are some common troubleshooting steps to help resolve them:

  • Ensure that you are using a compatible web browser. EAS-Tools is designed to work with modern browsers such as Google Chrome, Mozilla Firefox, and Microsoft Edge. Other browsers are untested, and may not function as expected. If you run into any issues, try switching to one of these supported browsers.
  • Clear your browser cache and cookies. Sometimes, cached data can cause issues with web applications. Clearing your cache and cookies can help resolve these problems.
  • Ensure JavaScript is enabled in your browser settings. EAS-Tools relies on JavaScript for its functionality, so make sure it is not blocked or disabled. Otherwise, it's pretty useless without JS enabled.
  • Check for any browser extensions or plugins that may interfere with EAS-Tools. Some extensions can block certain scripts or functionalities, so try disabling them temporarily to see if it resolves the issue. Ad blockers should not interfere normally with the functionality of EAS-Tools (there are no ads/trackers anywhere to speak of, anyhow...), but privacy-focused extensions may cause issues.
  • Ensure that your internet connection is stable. A weak or intermittent connection can cause issues with loading or using EAS-Tools. Once the tools are fully loaded in, you should be able to use all of them offline if you want, but a stable internet connection is recommended for the best experience.
  • If you are using the Decoder tool, ensure that your microphone is properly connected and configured. Check your browser's permissions to make sure it has access to your microphone. The audio coming in from the mic needs to be relatively clean to be processed correctly.
  • If you are using the Encoder/Splicer tools with Text-to-Speech announcements, ensure that you have a stable internet connection, as the TTS generation is performed on an external server.
  • Refer to the FAQ section above for answers to common questions and issues. You may find a solution to your problem there.
  • If you continue to experience any issues, consider reaching out for help and support. You can report bugs or request features by visiting the project's GitHub Issues page.

By following these fairly straightforward troubleshooting steps, you should be able to resolve most common issues encountered while using EAS-Tools. If you need further assistance, don't hesitate to seek help from the community or myself.
